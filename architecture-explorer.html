Two-tab interface:

Active items are highlighted in orange. On mobile (<768px), the sidebar collapses into a hamburger menu.

    Security Measures
    🔒 Signed URLs
    Files are never exposed publicly. Every view or download generates a temporary signed URL that expires in 1 hour. Share links get 24-hour URLs.
    🚦 Rate Limiting
    Upload endpoint is limited to 20 requests per minute per user, preventing abuse or accidental floods.
    🔑 Token Authentication
    Every API request requires a valid Clerk JWT token. The backend verifies it on every single request — no session cookies.
    🛡️ Double Protection
    Admin pages are protected twice: the frontend blocks non-admins from seeing the page, AND the backend rejects unauthorized API calls.
    📝 File Validation
    File types are validated both by extension and MIME type on the backend. Only accepted formats per document type are allowed.
    🎲 Share Tokens
    Share links use random UUIDs (like a lottery number). They can't be guessed and expire automatically after 24 hours.
    Data Flows
    How data moves through the system for each major action.

    📤 File Upload Flow

    1
    User selects files
    Drag-and-drop or file picker in the Upload Modal. Optionally add a custom name and tags. Frontend validates file types match the document category.
    2
    Backend receives & validates
    Checks rate limit (20/min), verifies user permission, validates file extensions against allowed types for this document category, confirms the company exists.
    3
    Stored in AWS S3
    File is uploaded to Amazon's cloud storage in Mumbai (India region). Stored at: companyId/docType/unique-id-filename. The filename is sanitized for safety.
    4
    Recorded in database
    A Document record is created with the file name, size, type, S3 location, tags, and uploader info. An audit log entry records who uploaded what.
    5
    Success feedback
    User sees a success notification. The document section refreshes to show the new file immediately.

    🔗 Sharing Flow

    1
    User clicks Share
    The Share Modal opens for the selected file.
    2
    Backend generates share link
    Creates a unique random token (UUID) and sets a 24-hour expiry. Saves to the document record in the database.
    3
    Modal shows share options
    Displays the share URL, a QR code, and buttons for WhatsApp, Email, and Copy Link. Shows "Expires in 24 hours".
    4
    Recipient opens link
    No login required. The public share page validates the token, checks it hasn't expired, generates a temporary download URL, and shows a download button.

    🐳 Docker Containers
    Both the frontend and backend run in Docker containers — isolated, reproducible environments that work identically in development and production.
    🚀 Render Hosting
    Deployed on Render.com with automatic builds. Two separate services: one for the Next.js frontend, one for the Express API. Connected to the same database.
    🔀 Nginx Proxy
    A reverse proxy sits in front, routing web traffic to the frontend and API traffic to the backend. Handles SSL, compression, and caching headers.
    ☁️ AWS S3 (Mumbai)
    All uploaded files live in Amazon S3, specifically in the Mumbai (ap-south-1) region for fast access from India. Files are stored privately; access is via signed URLs only.
    🗄️ PostgreSQL (Cloud)
    The database runs on a managed cloud PostgreSQL service (Neon or Railway). Stores all metadata — companies, documents, audit logs — but no actual files.
    📄 Swagger API Docs
    Interactive API documentation available at /api/docs. Developers can explore and test all endpoints directly from the browser.
